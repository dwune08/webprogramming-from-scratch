3.1 JavaScript의 시작

HTML은 웹 페이지의 구조를 정의하고, CSS는 그 구조에 스타일을 입히는 역할을 한다. 하지만 HTML과 CSS만으로는 사용자 입력 처리, 데이터 변경, 복잡한 상호작용과 같은 동적인 기능을 구현하는 데 한계가 있다. 이러한 동적인 기능은 주로 자바스크립트가 담당한다.

3.1.1 프로그래밍과 웹 코딩

자바스크립트는 HTML과 CSS로 만든 웹 페이지에 동적인 기능과 사용자 상호작용을 추가할 수 있게 해주는 웹 개발의 핵심 프로그래밍 언어이다.

별도의 컴파일 과정 없이 브라우저에서 실행할 수 있으며, 웹 코딩뿐만 아니라 일반적인 프로그래밍 사고와 논리적인 문제 해결 능력을 기르는 데도 도움이 된다.

3.1.2 자바스크립트의 주요 특징

자바스크립트의 주요 특징은 다음과 같다.

① 웹 개발을 위한 프로그래밍 언어

자바스크립트는 HTML, CSS와 함께 웹 페이지를 구성하는 핵심 기술 중 하나이며, 웹 페이지에 동적인 기능을 추가하는 역할을 한다.

② 스크립트 언어

자바스크립트는 브라우저에 내장된 자바스크립트 엔진을 통해 코드가 해석되어 실행된다. 이러한 특성 때문에 일반적으로 스크립트 언어로 분류된다.

③ 이벤트 기반 처리

사용자의 클릭, 입력, 스크롤과 같은 이벤트를 감지하고, 이벤트가 발생했을 때 미리 작성한 함수를 실행할 수 있다.

④ ECMAScript 표준

ECMAScript는 자바스크립트의 표준을 정의한 명세이다. 일반적으로 ES라고 줄여 부르며, ES6는 2015년에 발표된 대표적인 버전이다.

3.1.3 JavaScript 구현 방법

자바스크립트 코드는 HTML 문서 안에 직접 작성할 수도 있고, 별도의 .js 파일로 분리하여 작성할 수도 있다.

① inline 방식

HTML 태그의 이벤트 속성에 자바스크립트 코드를 직접 작성하는 방식이다.

<태그명 이벤트속성="실행할 자바스크립트 코드">
    화면에 표시할 내용
</태그명>

3.1.5 명령형 방식과 선언형 방식

같은 결과를 만들더라도 코드를 작성하는 방식은 다를 수 있다. 아래 예제는 배열에서 짝수만 추출하는 코드를 명령형 방식과 선언형 방식으로 비교한 것이다.

개념 정리

명령형 방식은 원하는 결과를 얻기 위해 어떤 순서로 처리할지 과정을 하나씩 작성하는 방식이다. 반복문, 조건문, 변수 변경 등을 직접 사용하여 처리 흐름을 구체적으로 작성한다.

선언형 방식은 처리 과정보다 어떤 결과를 얻을지에 초점을 두는 방식이다. 자바스크립트에서는 filter(), map(), forEach()와 같은 배열 메서드를 사용하여 코드를 더 간결하게 작성할 수 있다. 

const fruit = ['사과', '바나나', '딸기'];

// 명령형 방식
for(let i = 0; i < fruits.length; i++) {
    console.log(fruit[i]);
}

// 선언형 방식
fruits.forEach(function(fruit) {
    console.log(fruit);
});
