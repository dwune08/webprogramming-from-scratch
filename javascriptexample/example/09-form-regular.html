3.9 정규 표현식과 form 유효성 검사

정규 표현식은 문자열이 특정 규칙에 맞는지 확인할 때 사용한다. 회원가입, 주문서, 예약 양식처럼 사용자가 입력한 값을 검사할 때 자주 활용된다.

3.9.1 정규 표현식의 개념

정규 표현식(Regular Expression, 정규식)은 특정한 규칙이나 패턴을 가진 문자열을 찾거나 처리하기 위한 표현식이다. 문자열이 특정 조건을 만족하는지 검사하거나, 문자열의 일부를 추출 또는 치환할 때 사용한다.

정규 표현식은 특별한 의미를 가진 기호와 문자를 조합하여 작성한다. JavaScript에서는 보통 /정규표현식/ 형태의 리터럴 표기법으로 작성한다. 

- RegExp 생성자를 이용한 생성
const regExp1 = new RegExp("정규표현식");

- 정규 표현식 리터럴을 이용한 생성
const regExp2 = /정규표현식/;

정규 표현식 뒤에는 검색 방식을 지정하는 플래그(flag)를 추가할 수 있다.

  • g (Global) : 문자열 안에서 일치하는 모든 항목을 찾는다.
  • i (Ignore Case) : 영문 대소문자를 구분하지 않고 검사한다.
  • m (Multi Line) : 여러 줄 문자열에서 각 줄의 시작과 끝을 검사할 수 있다.

3.9.2 정규 표현식 메서드 확인하기

정규 표현식은 문자열에서 특정 패턴을 검사, 검색, 치환할 때 사용한다. 이때 정규 표현식 객체의 메서드와 문자열 메서드를 함께 사용할 수 있다.

regExp.test(문자열) 문자열이 정규 표현식의 패턴과 일치하는지 확인하여 true 또는 false를 반환한다.
regExp.exec(문자열) 문자열에서 정규 표현식과 일치하는 값을 찾아 배열 형태로 반환한다. 일치하는 값이 없으면 null을 반환한다.
문자열.match(정규식) 문자열에서 정규식과 일치하는 값을 찾아 배열 형태로 반환한다. 일치하는 값이 없으면 null을 반환한다.
문자열.replace(정규식, 대체값) 정규식과 일치하는 문자열을 다른 문자열로 치환한다.
문자열.search(정규식) 정규식과 일치하는 문자열의 위치를 반환한다. 일치하는 값이 없으면 -1을 반환한다. 
const text = "JavaScript 123 JavaScript";
const regExp = /JavaScript/;

regExp.test(text); // 결과: true

regExp.exec(text); // 결과: ["JavaScript"]

text.match(regExp);// 결과: ["JavaScript"]

text.replace(regExp, "JS"); // 결과: "JS 123 JavaScript"

text.search(regExp);// 결과: 0

3.9.3 자주 사용하는 메타 문자

메타 문자는 정규 표현식에서 특별한 의미를 가지는 문자이다. 입력값의 형식이나 반복 횟수를 검사할 때 사용한다.

a문자열 안에 a라는 문자가 있는지 확인
[abcd]a, b, c, d 중 하나의 문자
[^abc]a, b, c를 제외한 문자 한 글자
[0-9]0부터 9까지의 숫자 중 한 글자
[a-z]영어 소문자 중 한 글자
[A-Z]영어 대문자 중 한 글자
[A-Za-z]영어 대문자 또는 소문자 중 한 글자
[가-힣]완성형 한글 한 글자
^문자열의 시작
$문자열의 끝
\d숫자 한 글자, [0-9]와 같은 의미
\w영문자, 숫자, 밑줄(_) 중 한 글자
\s공백 문자
{n}앞의 문자가 n번 반복
{n,m}앞의 문자가 n번 이상 m번 이하 반복
{n,}앞의 문자가 n번 이상 반복
*앞의 문자가 0번 이상 반복
+앞의 문자가 1번 이상 반복
?앞의 문자가 0번 또는 1번 등장
.개행 문자를 제외한 임의의 한 문자
()여러 패턴을 하나의 그룹으로 묶을 때 사용
\.마침표(.) 자체를 의미한다. 메타 문자를 문자 그대로 사용할 때는 \를 붙인다.

(?= ... )는 현재 위치 뒤쪽에 특정 패턴이 있는지 확인하는 조건 검사이다.
.*는 임의의 문자(.)가 0개 이상(*) 올 수 있음을 의미한다.

예를 들어 (?=.*[A-Za-z])는 임의의 문자가 0개 이상 나온 뒤 영문자(A-Z 또는 a-z)가 최소 1개 이상 포함되어 있는지 확인한다. 따라서 문자열의 어느 위치에 있든 영문자가 포함되어 있으면 조건을 만족한다.

(?=.*[A-Za-z]) : 영문자(A-Z 또는 a-z)를 최소 1개 이상 포함해야 한다.
(?=.*\d) : 숫자(0-9)를 최소 1개 이상 포함해야 한다.
(?=.*[@$!%*?&]) : 특수문자(@, $, !, %, *, ?, &)를 최소 1개 이상 포함해야 한다.
[A-Za-z\d@$!%*?&]{6,10} : 영문자, 숫자, 지정된 특수문자로 구성된 6글자 이상 10글자 이하의 문자열이어야 한다.

3.9.6 아이디 유효성 검사

아이디는 첫 글자를 영문자로 입력해야 하며, 첫 글자 이후에는 영문자와 숫자만 사용할 수 있다. 전체 길이는 5글자 이상 8글자 이하로 입력해야 하며, 특수문자는 사용할 수 없다. 

const idRegExp = /^[A-Za-z][A-Za-z0-9]{4,7}$/;

3.9.7 비밀번호 유효성 검사

비밀번호는 영문자, 숫자, 특수문자(@$!%*?&)를 각각 하나 이상 포함해야 하며, 전체 길이는 6글자 이상 10글자 이하로 입력해야 한다. 

const passwordRegExp = /^(?=.*[A-Za-z])(?=.*\d)(?=.*[@$!%*?&])[A-Za-z\d@$!%*?&]{6,10}$/;

const emailRegExp = /^[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Za-z]{2,}$/;
